[#ai-factory-sappco]
== How do I access images from {sappco}?

. Visit link:https://apps.rancher.io/[{sappco}], sign in, create and save the user access token as described in the link:https://docs.apps.rancher.io/get-started/authentication/[official documentation].
. Add the {sappco} credentials to the {ranchera} _local_ cluster.
.. From the main left panel, select menu:local[Storage > Secrets].
.. Select btn:[Create] in the top right and select btn:[Opaque].
+
[#fig-aif-rancher-secrets-opaque]
.{ranchera} secrets types
image::aif-rancher-secrets-opaque.png[Screenshot showing list of {ranchera} secrets types,scaledwidth=50%]
+
.. Select the btn:[aif-operator] namespace and enter your preferred secret name (referenced as `<APPCO_SECRETS_NAME>` in command line tasks) to identify the resource.
.. In the btn:[Data] tab, add the key `user` with the value of your {sappco} username and `token` with your {sappco} token.
+
[#fig-aif-rancher-secrets-keys]
.{sappco} secrets
image::aif-rancher-secrets-keys.png[Screenshot showing how to add secrets to {sappco},scaledwidth=50%]
+
.. Confirm by selecting btn:[Create] in the bottom right corner.

+
[TIP]
.Creating the secrets on the command line
====
[source,bash,subs="+attributes"]
----
{prompt_user}kubectl create secret generic <APPCO_SECRETS_NAME> \
  --from-literal=user=<APPCO_USERNAME> \
  --from-literal=token=<APPCO_USER_TOKEN> \
  -n aif-operator
----
====
. Assign {sappco} secrets to {saifactory}.
.. From the main left panel, select menu:SUSE AI Factory[Settings].
.. In the `SUSE Application Collection` section, select the secret name corresponding to your `<APPCO_SECRETS_NAME>` from the `Secret name` drop-down list.
.. Select `user` for menu:Username Secret[Secret Key] and `token` for menu:Access Token Secret[Secret Key].
+
[#fig-aif-secrets-sappco]
.{sappco} secrets
image::aif-secrets-sappco.png[Screenshot showing how to assign secrets for {sappco},scaledwidth=50%]
+
.. Confirm by selecting btn:[Apply] in the bottom right corner.
. Navigate to menu:SUSE AI Factory[Apps] and select btn:[SUSE AI Library] from the drop-down list in the top center of the screen.
. Select the btn:[Refresh] button in the top right.
The list of available applications from the {sappco} appears.

[#ai-factory-sregistry]
== How do I access images from {sregistry}?

. Visit link:https://scc.suse.com/[{scc}], select the organization with an active {productname} subscription from the left sidebar, and save the registration code for the {productname} subscription.
. Add the {sregistry} credentials to the {ranchera} _local_ cluster.
.. From the main left panel, select menu:local[Storage > Secrets].
.. Select btn:[Create] in the top right and select btn:[Opaque].
.. Select the btn:[aif-operator] namespace and enter your preferred secret name (referenced as `<REGISTRY_SECRETS_NAME>` in command line tasks) to identify the resource.
.. In the btn:[Data] tab, add the key `user` with the value `regcode` and `token` with your {sregistry} token.

[#ai-factory-fleet-gitops]
== How do I access a Git repository to store applications configuration?

. Add your Git Personal Access Token (PAT) to the {ranchera} _local_ cluster.
.. From the main left panel, select menu:local[Storage > Secrets].
.. Select btn:[Create] in the top right and select btn:[Opaque].
.. Select the btn:[aif-operator] namespace and enter your preferred secret name (referenced as `<GIT_SECRETS_NAME>` in command line tasks) to identify the resource.
.. In the btn:[Data] tab, add the key `token` and paste your PAT as a value.
+
[#fig-rancher-secrets-git]
.Git secrets
image::aif-rancher-secrets-git.png[Screenshot showing how to add secrets for Git,scaledwidth=50%]
+
.. Confirm by selecting btn:[Create] in the bottom right corner.

+
[TIP]
.Creating Git secrets on the command line
====
[source,bash,subs="+attributes"]
----
{prompt_user}kubectl create secret generic <GIT_SECRETS_NAME> \
  --from-literal=token=<GIT_PAT_TOKEN> \
  -n aif-operator
----
====
. Add the Git PAT to {saifactory}.
.. From the main left panel, select menu:SUSE AI Factory[Settings].
.. In the `Fleet/GitOps` section, enter the Git repository URL and specify which branch to use.
.. Select the authentication type and the PAT secret name corresponding to your `<GIT_SECRETS_NAME>` from the `Credential Secret` drop-down list.
.. Select `token` for menu:Secret Key.
+
