[#ai-factory-deployment-airgapped-workflow-downloading-artifacts]
== Downloading artifacts

In a connected environment, download all the necessary artifacts to the local data store.

=== {helm} charts and containers

For {helm} charts and container images, use any registry mirroring tool, such as regsync, {skopeo}, crane, {helm}, {docker}, or {podman}.
Refer to the documentation of the specific tool for instructions on downloading the artifacts to the local data store.

For example, to download the {owui} {helm} chart from the {sappco} OCI registry:

[source,shell,subs="+attributes"]
----
{prompt_user}helm pull --username <SUSE app username> --password <SUSE application password> oci://dp.apps.rancher.io/charts/open-webui --version 8.19.1
----

To pull the {owui} container image from the {sappco} OCI registry:

[source,shell,subs="+attributes"]
----
{prompt_user}docker login dp.apps.rancher.io
{prompt_user}docker pull dp.apps.rancher.io/containers/open-webui:0.6.41-14.80
----

[TIP]
====
For mirroring container images, we recommend using the `SUSE-AI-get-images.sh` and `SUSE-AI-load-images.sh` scripts from the link:https://github.com/SUSE/suse-ai-stack/tree/main/air-gapped-install[{suseai} air-gapped] repository.
====

=== Models

To cache {nvidia} NIM or {huggingface} models, refer to the link:https://docs.nvidia.com/nim/large-language-models/latest/deployment/air-gap-deployment.html#air-gap-deployment[NVIDIA NIM air-gap deployment documentation].

[#ai-factory-deployment-airgapped-workflow-transfer-data]
== Make downloaded data available to the air-gapped environment

Make the downloaded data available inside the air-gapped environment using `copy`, `rsync`, or another suitable data transfer method.

[#ai-factory-deployment-airgapped-workflow-setup-registries]
== Set up OCI-compliant registries in the air-gapped environment

Ensure one or more OCI-compliant private registries are available inside the air-gapped environment to mirror the artifacts.
If your organization does not have an existing registry, you can use any commercially supported option, such as link:https://apps.rancher.io/applications/harbor[Harbor].
Refer to the documentation of the specific registry you plan to deploy for more details.
We strongly recommend configuring endpoint protection with production TLS certificates.

[#ai-factory-deployment-airgapped-workflow-setup-registries-access]
=== Configure {productname} to access image registries?

After air-gapped offline registries exist, configure {productname} to access them by specifying endpoints for their repositories:

. From the main left panel, click menu:SUSE AI Factory[Settings].
. In the menu:Advanced[Registry Endpoints] section, enter endpoint URLs for local {sappco}, {sregistry} or {nvidia} registry.
. Confirm with btn:[Apply] in the bottom right corner.

[#ai-factory-deployment-airgapped-workflow-upload-artifacts]
== Upload {helm} charts and containers

Use any registry mirroring tool, such as regsync, {skopeo}, crane, {helm}, {docker}, or {podman}, to upload the {helm} charts and container images to your local private registries.
Refer to the documentation of the specific tool for instructions on uploading the artifacts.

For example, to upload the {owui} {helm} chart to your local registry:

[source,shell,subs="+attributes"]
----
{prompt_user}helm push \
  --username <local registry username> \
  --password <local registry password> \
  open-webui-8.19.1.tgz oci://my.local.registry.com/suse-app-co/charts
----

To tag and push the {owui} container image to your local registry:

[source,shell,subs="+attributes"]
----
{prompt_user}docker tag dp.apps.rancher.io/containers/open-webui:0.6.41-14.80 my.local.registry.com/suse-app-co/containers/open-webui:0.6.41-14.80
{prompt_user}docker push my.local.registry.com/suse-app-co/containers/open-webui:0.6.41-14.80
----

[#ai-factory-deployment-airgapped-workflow-create-pvc]
== Create a PVC for the model cache

Refer to the link:https://kubernetes.io/docs/concepts/storage/persistent-volumes/#persistentvolumeclaims[Kubernetes PVC documentation] on how to create a PVC with ReadWriteMany access mode.

[#ai-factory-deployment-airgapped-workflow-transfer-cache]
== Transfer model cache to the PVC

There are multiple ways to transfer data from a local store into the PVC.
For example, you can create a simple job to mount both the local data store and the PVC as volumes, and then copy the data over from inside the pod.

[#ai-factory-deployment-airgapped-workflow-configure-charts]
== Configure {helm} charts to use the model cache PVC

Configuring the {nvidia} Blueprint RAG to use an existing PVC is not currently supported.
See link:https://github.com/NVIDIA-AI-Blueprints/rag/issues/693[] for more details.

[#ai-factory-deployment-airgapped-workflow-configure-rke2]
== Configure {rke2a} private registries

Follow the link:https://documentation.suse.com/cloudnative/rke2/latest/en/install/private_registry.html[RKE2 private registry configuration documentation] to configure the registry mirrors.
If your local private registries use prefix paths to namespace the artifacts, use link:https://documentation.suse.com/cloudnative/rke2/latest/en/install/private_registry.html#_rewrites[rewrite rules].
You must configure one entry per mirror.
Perform the private registry configuration on all nodes in all clusters inside the air-gapped environment.
