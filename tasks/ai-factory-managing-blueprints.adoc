[#ai-factory-managing-blueprints-create]
== How do I create custom blueprints?

. From the main left panel, select menu:SUSE AI Factory[Blueprints > Create].
. On the `Basic Info` screen, enter the name, version, and optional description of the blueprint.
+
[#fig-aif-bps-create-basic]

image::aif-bp-create1.png[Screenshot showing basic information about a blueprint,scaledwidth=50%]
+
. On the `Select Apps` screen, search for an application to add to the template, select its name, and specify the version to install.
Repeat this step for all applications you want to include in the blueprint.
+
[#fig-aif-bps-create-selapps]
.Selecting applications of a blueprint
image::aif-bp-create2.png[Screenshot showing how to add applications to a blueprint,scaledwidth=50%]
+
. On the `Configuration` screen, review and optionally update the {helm} deployment chart of the blueprint.
+

.Custom applications configuration
image::aif-bp-create3.png[Screenshot showing how to modify configuration of applications belonging to a blueprint,scaledwidth=50%]

[#ai-factory-managing-blueprints-modify]
== How do I modify blueprints?

. From the main left panel, select menu:SUSE AI Factory[Blueprints].
. From the list of existing templates, select the blueprint to modify, open its three-dot action menu, and select btn:[Edit].
+
[#fig-aif-bps-edit]
.Modify blueprints
image::aif-bp-edit.png[Screenshot showing how to modify a blueprint,scaledwidth=50%]
+
. On the `Basic Info` screen, update the version number and description.
. On the `Select Apps` screen, update the list of applications in the blueprint by adding new applications or removing existing ones.
Confirm with btn:[Next].
. On the `Configuration` screen, review and optionally update the {helm} charts of the blueprint applications.
. On the `Review & Create` screen, verify that the entered information is correct and confirm with btn:[Save as New Version].

[#ai-factory-managing-blueprints-copy]
== How do I clone a blueprint?

. From the main left panel, select menu:SUSE AI Factory[Blueprints].
. From the list of existing blueprints, select the template you want to clone, specify the source version, open its three-dot action menu, and select btn:[Copy].
+
[#fig-aif-bps-copy]
.Clone blueprints
image::aif-bp-copy.png[Screenshot showing how to clone a blueprint,scaledwidth=50%]
+
. On the `Basic Info` screen, you can modify the suggested name, version and description.
. On the `Select Apps` screen, update the list of applications in the blueprint by adding new applications or removing existing ones.
Confirm with btn:[Next].
. On the `Configuration` screen, review and optionally update the {helm} deployment chart of the blueprint.
. On the `Review & Create` screen, verify that the entered information is correct and confirm with btn:[Create].

[#ai-factory-managing-blueprints-deprecate]
== How do I deprecate blueprints?

. From the main left panel, select menu:SUSE AI Factory[Blueprints].
. From the list of existing blueprints, select the template to deprecate.
. Specify the version to deprecate, open its three-dot action menu, and select btn:[Deprecate].
+

[#ai-factory-managing-blueprints-undeprecate]
== How do I restore deprecated blueprints?

. From the main left panel, select menu:SUSE AI Factory[Blueprints].
. A deprecated blueprint is hidden from the list of blueprints.
To make it visible again, check the btn:[Show deprecated] box.
. Select the blueprint you want to restore and specify the deprecated version.
. Open the three-dot action menu for the blueprint, and select btn:[Undeprecate].
. Confirm with btn:[Undeprecate].

[#ai-factory-managing-blueprints-delete]
== How do I delete blueprints?

. From the main left panel, select menu:SUSE AI Factory[Blueprints].
. From the list of existing templates, select the blueprint to delete and specify the target version.
. Open the three-dot action menu for the blueprint, select btn:[Delete], and confirm the deletion.
+

[#ai-factory-managing-blueprints-multiple-envs]
== How do I manage environment-specific configurations with blueprints?

Consider the following use-case:
You want to deploy a single `{litellm}` blueprint to both a development cluster and a production cluster, but with different configurations. 
Because of resource constraints on development nodes, you must configure different LLM models, {ingress} settings, or GPU allocations.
You can achieve this by managing separate blueprints named `blueprint-litellm-dev` and `blueprint-litellm-prod`.

To configure and manage environment-specific deployments, perform the following steps:

. Use a prebuilt blueprint or create a custom blueprint as described in xref:ai-factory-managing-blueprints-create[].
Because this template serves the production cluster, name the blueprint `blueprint-litellm-prod`.
. Create a new version of the blueprint with the desired configurations as described in xref:ai-factory-managing-blueprints-modify[].
. Clone the blueprint as described in xref:ai-factory-managing-blueprints-copy[] and name it `blueprint-litellm-dev`.
. Deploy the blueprints as described in xref:ai-factory-managing-blueprints-install[]. Specify the correct version for each deployment.
.. Deploy `blueprint-litellm-prod` to the production cluster.
.. Deploy `blueprint-litellm-dev` to the development cluster.

[TIP]
====
To apply a new blueprint version during future maintenance window upgrades, select menu:SUSE AI Factory[Workloads] and select the corresponding btn:[Upgrade] button.
====
