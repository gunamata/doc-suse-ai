[abstract]
--
{productname} is a {ranchera} user interface (UI) extension and a {kube} operator.
It replaces manual, fragmented application integration with pre-validated, version-controlled blueprints.
This establishes a standardized and reproducible pipeline for deploying and scaling complex AI workloads on {kube} clusters.

ifeval::["{PROF_PRODUCT}" == "saifnv"]
{productname} lets you install and manage {nvidia} applications and blueprints from {nvidia} GPU Cloud (NGC) in a {ranchera} environment.
endif::[]
--

== How is {productname} different from {suseai}?

{suseai} is the complete, sovereign AI solution, while {productname} is the component that builds and manages AI applications and blueprints on the platform.
The relationship can be summarized as: `Infrastructure Platform (like {rancherprime}) + Application Platform ({productname}) = {suseai}`.

While they are part of the same ecosystem, they serve different functions:

.{suseai} vs. {productname}
[cols="1,2a,2a"]
|===
| |*{suseai}* |*{productname}*

|*Core Focus*
|Infrastructure and security
|AI application deployment and lifecycle management

|*Primary Goal*
|Provides a secure, vendor-agnostic foundation to run and govern AI workloads.
|Standardizes and automates the deployment of AI applications using preconfigured templates.

|*Operational Role*
|Acts as the stable and manageable environment (the "factory floor").
|Provides the tools to build, deploy, and manage AI workloads (the "assembly line").

|*NVIDIA Support*
|Does not directly enable or manage {nvidia} applications.

== Why use {productname}?

{productname} provides a solution that replaces manual integration of AI applications with pre-validated blueprints.
It eliminates inconsistent "snowflake" environments and enforces governance and auditability through centralized controls.

Running individual AI applications often forces teams to manually stitch together multiple open-source tools using fragmented {helm} charts.
This "do-it-yourself" approach means that teams spend excessive time on integration and maintenance rather than building AI value.

{productname} replaces this manual burden with a digital assembly line that offers several advantages:

ifeval::["{PROF_PRODUCT}" == "saif"]
Pre-validated blueprints::
endif::[]
ifeval::["{PROF_PRODUCT}" == "saifnv"]
Pre-validated {nvidia} blueprints::
endif::[]
Instead of building complex architectures from scratch, {productname} uses blueprints. These are curated, opinionated "master templates" that bundle multiple apps together and deploy everything needed for a specific use-case (like Retrieval-Augmented Generation or MLOps) in a single action.

Elimination of "snowflake" environments::
Individual deployments frequently result in inconsistent, fragile setups that are difficult to replicate. {productname} standardizes your infrastructure by treating blueprints as immutable, version-controlled reference stacks. When you deploy a workload via {productname}, it strictly records its exact source provenance, allowing platform teams to seamlessly roll out upgrades or deprecate older versions without relying on informal tracking or spreadsheets.

Built-in governance and auditability::
Reproducing individual, manual deployments is often "detective work" with informal governance.
{productname} provides centralized governance with zero-trust security guardrails and continuous vulnerability scanning to support secure and compliant AI deployments.
This helps organizations maintain an auditable AI stack and meet regulatory requirements such as the EU AI Act.

Reduced maintenance burden::
Because {suse} handles the underlying component selection, integration testing, security patching and lifecycle management, platform teams can reduce their ongoing maintenance burden.

== What are the building blocks of {productname}?

{productname} consists of a {kube} operator and a {ranchera} UI extension.
It lets you install and manage individual applications from {ailibrary} as well as group them into stacks called _blueprints_.

[#ai-factory-foundation]
=== Foundation
The base layer of {productname} is deployed on bare metal or cloud-hosted nodes, preferably deployed by the {ranchelemental}.
It consists of the following components:

* a prescribed configuration of the {slsa} 16 operating system
* {rke2a} {kube} management cluster
* {ranchermanager} that supports the UI extension and {kube} operator code for {productname}
* optionally, {ssecurity} and {sobservability} work together to provide high security and advanced monitoring capability across all AI workloads

[#ai-factory-itself]
=== AI Factory
{productname} is a {ranchera} UI extension that serves the following purposes:

ifeval::["{PROF_PRODUCT}" == "saif"]
* Discover, install and manage individual applications from {ailibrary}.
* Discover, install and manage preconfigured blueprints that bundle specific applications to match a target use-case.
endif::[]
ifeval::["{PROF_PRODUCT}" == "saifnv"]
* Discover, install and manage individual applications from the {nvidia} GPU Cloud (NGC) catalog.
endif::[]
* Create custom blueprints based on customer's AI application selection.

Both running applications and blueprints can be shared across {ranchera}-managed clusters.

[#ai-factory-blueprints]
=== Blueprints
Blueprints are published, immutable and version-controlled stacks of applications required for a specific use-case.
Because AI workloads require multiple interconnected applications, a Blueprint tightly packages these necessary components into a single unit that can be deployed with one click.

An example of a Blueprint is a RAG (Retrieval-Augmented Generation) workload.
It typically consists of the following components:

* A *vector database* such as {qdrant} or {milvus}, to store high-dimensional vector embeddings.
* An *LLM inference engine* such as {ollama} or {vllm} to process both the user's prompt and the retrieved context to generate the final textual response.
* A *user interface* such as {owui} to provide an interface for end-users to interact with the pipeline.

ifeval::["{PROF_PRODUCT}" == "saif"]
{productname} is delivered with a set of prebuilt {suse} blueprints.
endif::[]
ifeval::["{PROF_PRODUCT}" == "saifnv"]
{productname} is delivered with a set of prebuilt {nvidia} blueprints.
endif::[]
You can also create custom blueprints and select your own set of AI applications.

[#ai-factory-manager-roles]
== What are the workflows and responsibilities?

The workflow is defined by a user role and is different for a user and an administrator.

  * *Admin Users* install and configure the extension.
  * *Cluster Owners* and *{ranchera} Admin Users* discover, deploy and manage AI applications and blueprints from {suse} image registries, such as {sappco} and {sregistry}, and from the {nvidia} GPU Cloud (NGC) catalog.
